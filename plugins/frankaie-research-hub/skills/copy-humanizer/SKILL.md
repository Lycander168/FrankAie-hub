# 文案去 AI 味潤稿器（Copy Humanizer）

這是一個**工具型 skill**。功能：把讀起來「像 AI 寫的」文案，改成自然、有溫度、貼品牌語氣的版本，同時**不偏離原意、不誇大**。
和 `marketing-team` / `listing-optimizer` / `ad-hook-writer` 搭配使用。

> 原則：自然 ≠ 隨便。去掉的是公式化套路，保留的是清楚、誠實與品牌個性。

---

## 何時用

- 文案讀起來生硬、像範本、AI 味重
- 要統一不同素材的品牌語氣
- 上架前最後潤稿、提升可讀性與信任感

## 需要的輸入

1. 要潤的文案（中文 / 英文 / 中英混）
2. 用途與平台（Listing / 廣告 / 社群 / EDM）
3. 品牌語氣（可選：專業沉穩 / 親切日常 / 年輕活潑）
4. 必須保留的硬資訊（規格 / 數字 / 法規用語）

## 它會處理的「AI 味」特徵

- 過度對仗 / 排比的句式（「不僅…更…」連發）
- 萬用空話（「在當今快節奏的世界」「極致體驗」）
- 機械化轉折詞（furthermore / moreover 連用）
- 每段都「總分總」的範本結構
- 形容詞堆砌但沒有具體畫面
- 中英直譯腔（被動句過多、名詞化過重）

## 運作流程

1. **原意鎖定**：先標出不可動的硬資訊（規格 / 數字 / 法規），潤稿時不碰。
2. **去套路**：移除空話與公式句，換成具體、口語、有畫面的說法。
3. **對齊語氣**：依品牌 tone 調整人稱、節奏、用詞。
4. **可讀性**：拆長句、減被動、加節奏感。
5. **誠實檢查**：確認沒有為了「好聽」而新增未經證實的主張。
6. **附改動清單**：列出主要改動與理由，方便團隊複用原則。

## 輸出範本

```
# 潤稿結果 | <用途/平台>｜<日期>

## 潤後文案
<改寫後版本>

## 改動清單（改了什麼 / 為什麼）
| 原句（節錄） | 改為 | 原因 |
|--------------|------|------|

## 保留未動（硬資訊）
- <規格/數字/法規用語清單>

## 風格備註
- 本次採用語氣：
- 後續同類文案建議遵循的 2–3 條原則：
```

## 實例演練（Worked Example）

**原文（AI 味重）**：「在當今快節奏的生活中，我們的 65W GaN 充電器為您帶來極致的充電體驗，不僅快速更兼具安全，是您的不二之選。」

**潤後**：「出門只想帶一顆充電器？這顆 65W GaN 筆電、手機一起充，包包裡不卡位、用起來也不燙手。」

**改動清單（節錄）**：
| 原句 | 改為 | 原因 |
|------|------|------|
| 在當今快節奏的生活中 | （刪除） | 萬用空話、無資訊 |
| 極致的充電體驗 | 一起充、不燙手 | 換成具體可感知的利益 |
| 不二之選 | （刪除） | 公式化套話 |

**保留未動**：65W、GaN（硬規格不改）。

## 與團隊串接

- **上游**：初稿可來自 `listing-optimizer`、`ad-hook-writer`、`marketing-team`。
- **平行**：可丟 `consumers-*` 小組確認「讀起來可信、想買嗎」。
- **下游**：定稿交 `marketing-team` / `ecommerce-operator` 使用。

> 提醒：去 AI 味是改「表達」不是改「事實」。任何規格 / 功效 / 認證主張保持原樣，不可為了通順而新增或放大。
