# AI 消費者驗證關卡（Validation Panel）

這是 HUB 的**招牌驗證關卡**：用一套統一標準，召集 4 個市場的 AI 消費者小組，
對「產品概念 / 定價 / 文案 / 視覺」做低成本、可比較的購買意願驗證。

> 心法：在投錢前，先用模擬受眾排除明顯的壞假設。便宜、快、可比較——但不取代真人市場驗證。

> 定位：本 skill 是 `hub-router` 標準動線中的**固定「驗證閘」**；三階段 SOP 的 Stage 2 預設經過它。

---

## 何時啟動
- 任一階段要回答「這個會不會被買 / 哪個版本較好 / 定價可不可接受」。
- `hub-router` 動線的立項(1) 與上市前(5) 驗證關卡。
- 使用者直接說「測一下 / 打分 / 召集焦點小組」。

## 統一購買意願量表（1–5 Rubric）

| 分數 | 定義 |
|------|------|
| 5 | 一定買，會主動推薦 |
| 4 | 會買，少量顧慮 |
| 3 | 猶豫，需更多資訊 / 誘因 |
| 2 | 偏不買，明顯阻力 |
| 1 | 不會買 |

> 每位 persona 給「分數 + 一句理由 + 最大顧慮」；小組給「平均分 + 分數分佈」。
> 判讀基準（預設）：平均 ≥3.8 可推進；3.0–3.8 需改善再測；<3.0 重新檢視。

## 標準驗證流程
1. **定義受測物與問題** — 測什麼（概念/價/文案/視覺）、要回答什麼決策。
2. **選市場** — 依目標市場召集對應 `consumers-*`（可單一或多市場並行）。
3. **統一評分** — 各 persona 依 1–5 rubric 給分 + 理由 + 顧慮。
4. **彙整** — 平均分、分佈、共識點、主要異議、改善建議。
5. **跨市場對比**（多市場時）— 比較各市場接受度與差異化顧慮。
6. **結論** — 推進 / 改善再測 / 退回，附信心度。

## 與 LYCANDER GROUP 的協作介面
- **調度**：召集 `consumers-taiwan` / `consumers-usa` / `consumers-japan` / `consumers-europe` 產生反應，本關卡統一評分與彙整。
- **上游**：受測物可來自 `product-manager`（概念）、`pricing-calculator`（定價）、`listing-optimizer`/`copy-humanizer`（文案）、`visual-team`/`animation-designer`（視覺）。
- **下游**：結論回 `hub-router` 決定推進與否、回 `product-manager` 修規格；驗證紀錄寫進 `planning-with-files` 的 `findings.md`、交付前過 `quality-gate`。

## 啟動方式
收到「驗證 / 會不會買 / 打分」需求時：定義受測物與決策問題 → 選市場 → 套 1–5 rubric →
產出報告（多市場則附跨市場對比）→ 給推進/改善/退回結論。

> 提醒：本關卡產出為**虛擬受眾模擬推估，非真實市場數據**；
> 平均分與結論僅供方向驗證與假設排序，重大投資決策仍須以真人市場調查／實測佐證。
