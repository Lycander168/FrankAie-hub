# 評論痛點萃取器（Review Miner）

這是一個**工具型 skill**。功能：把一堆雜亂的評論 / Q&A / 退貨原因，整理成**量化、可行動**的洞察：客戶最痛什麼、最愛什麼、我們的切入機會在哪。
是 `electronics-team`、`marketing-team`、`ecommerce-operator`、`cx-team` 的共用情報來源。

> 原則：洞察要可量化、可行動。不是「大家覺得不錯」，而是「38% 一星集中在發熱，對應我們的散熱賣點」。

---

## 何時用

- 立項前分析競品評論，找市場縫隙
- 自家產品上市後，從評論 / 退貨找改善點
- 寫 Listing / hook 前，確認真正的痛點與賣點機會

## 需要的輸入

1. 評論文字（貼上，或競品連結 + 你能取得的內容）
2. 來源與情境（哪個平台、競品還是自家、星等分布若有）
3. 想回答的問題（找改善點 / 找賣點 / 找定價痛點…）

## 運作流程

1. **分類編碼**：把每則評論歸到主題（功能 / 品質 / 外觀 / 價格 / 物流 / 客服 / 安全）。
2. **情緒分層**：正面 / 中性 / 負面，並抓「強烈詞」與「重複詞」。
3. **量化**：每個痛點 / 讚點算出現頻率（% 或次數），排 Top N。
4. **根因推測**：把表面抱怨對映到可能的工程 / 設計 / 營運根因（標為推測）。
5. **轉行動**：每個洞察給「對應動作」——產品改善、賣點主打、或文案切角。

---

## 與團隊串接

- **下游（產品）**：痛點根因交 `electronics-team` / `mechanical-team` 改善。
- **下游（行銷）**：賣點機會交 `ad-hook-writer` / `listing-optimizer` / `marketing-team`。
- **下游（服務）**：客訴模式交 `cx-team` 優化售後。
- **驗證**：洞察可丟 `consumers-*` 小組確認「是不是真痛點」。

> 提醒：頻率與根因若樣本不足或來源偏誤，明確標 `【需查證】`，不把少數聲音當全體共識。
