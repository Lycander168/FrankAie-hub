# Google 雲端硬碟 Markdown 報告瀏覽器（Drive MD Browser）

這是一個**工具型 skill**。HUB 內各團隊（客服、營運、競品、供應鏈、產品、視覺、AI 情報…）每天會把工作成果以 **`.md` 格式**寫進 Google 雲端硬碟。這個 skill 負責把那一堆報告**找出來、列清楚、讀進來、彙整好**，讓使用者不必自己翻資料夾。

> 原則：先定位再讀取，讀取後一定附「來源檔名＋連結＋修改時間」。看不到內容不臆造、找不到檔案先回報搜尋條件，不假裝看過。

---

## 何時用

- 想知道「雲端上有哪些報告 / 最新一份是哪個」
- 要打開某一份特定報告（依名稱、日期或類型）來讀內容
- 要把同一類報告（例如連續幾天的客服監控）**彙整 / 比對趨勢**
- 交接前要列一份「今天團隊產出了哪些 .md」的索引

> 若使用者要的是「產生新報告」而非「瀏覽既有報告」，這不是本 skill 的工作——交回對應的角色團隊（cx-team / operations-team…）去產出。

## 需要的輸入（缺哪項就問或用預設）

1. **範圍**：全部？某資料夾？某類型（客服 / 營運 / 競品…）？某段日期？
2. **動作**：只列索引、讀某一份、還是彙整多份？
3. **排序 / 數量**：預設依「最近修改」由新到舊，預設列 10–20 筆。

## 可用的 Google Drive 工具（對應動作）

| 動作 | 工具 | 重點參數 |
|------|------|----------|
| 找 / 列 .md | `search_files` | `query: "title contains '.md' or mimeType = 'text/markdown'"`，可加 `modifiedTime > '...'`、`parentId = '...'`、`title contains '客服'` |
| 看最近檔案 | `list_recent_files` | `orderBy: recency / lastModified`，`pageSize` |
| 讀內容 | `read_file_content` | `fileId`，需要留言再加 `includeComments: true` |
| 查單檔資訊 | `get_file_metadata` | `fileId`（要確認大小 / 擁有者 / 修改時間時） |

> 分頁：結果有 `nextPageToken` 就代表還有；要看更多就帶 `pageToken` 續抓，別只憑第一頁就說「總共只有這些」。

## 運作流程

1. **定位**：依使用者的範圍組 `search_files` 查詢（類型→`title contains '關鍵字'`；日期→`modifiedTime` 區間；資料夾→`parentId`）。先抓檔名清單，**不要急著讀全文**。
2. **列索引**：把命中的檔案整成索引表（檔名 / 類型 / 修改時間 / 大小 / 連結），讓使用者挑。
3. **讀取**：使用者挑定後（或只命中一份時）用 `read_file_content` 讀進來。
4. **摘要 / 彙整**：單份→抓重點與行動項；多份→沿同一維度比對（趨勢、變化、未結項）。
5. **附來源**：每條結論標來源檔名與 `viewUrl`，無法讀取或為空檔（size 0）的明確標註，不腦補內容。

## 輸出範本

### A. 索引模式（列出有哪些報告）

```
# 雲端報告索引 | <範圍>｜查詢時間 <日期>

查詢條件：<title/類型/日期/資料夾>　命中 N 筆（是否還有下一頁：是/否）

| # | 報告名稱 | 類型 | 修改時間 | 大小 | 連結 |
|---|----------|------|----------|------|------|
| 1 | 客服監控報告_2026-06-19_v3.md | 客服 | 06-19 00:30 | 28KB | <viewUrl> |
| 2 | 營運晨報_2026-06-19.md | 營運 | 06-19 00:29 | 22KB | <viewUrl> |

下一步：要我打開哪一份，或彙整其中某幾份？
```

### B. 單份閱讀模式

```
# 報告摘要 | <檔名>
來源：<viewUrl>｜修改：<時間>｜擁有者：<owner>

## 重點摘要
- …（忠實摘錄，不加料）

## 行動項 / 待結案
- …

## 數字 / 異常（若有）
- …（無法從原文確認的標 【需查證】）
```

### C. 彙整模式（多份比對趨勢）

```
# 報告彙整 | <類型>｜<日期區間>｜來源 N 份

## 趨勢/變化
| 維度 | 最早(<檔名>) | 最新(<檔名>) | 變化 |
|------|--------------|--------------|------|

## 持續未結 / 反覆出現的問題
- …（標出現在哪幾份）

## 來源清單
- <檔名1> <viewUrl> ／ <檔名2> <viewUrl> …
```

## 實例演練（Worked Example）

**輸入**：「列出雲端硬碟今天最新的幾份報告，客服那份幫我打開。」

**步驟**：
1. `search_files`，`query: "title contains '.md' and modifiedTime > '2026-06-19T00:00:00Z'"`，依修改時間排序 → 得到 `AI產業每日情報_2026-06-19.md`、`客服監控報告_2026-06-19_v3.md`、`營運晨報_2026-06-19.md`…
2. 用「索引模式」列表給使用者挑。
3. 客服那份取其 `fileId` → `read_file_content` → 用「單份閱讀模式」回摘要＋行動項＋來源連結。
4. 順手提示：要不要把這份客服報告丟給 `review-miner` 萃取痛點。

**常見組合查詢**：
- 某類型最新版：`title contains '客服監控報告'`（再依 `modifiedTime` 取最新）
- 某資料夾全部：`parentId = '<資料夾ID>' and title contains '.md'`
- 近 7 天所有報告：`title contains '.md' and modifiedTime > '<7天前ISO>'`

## 與團隊串接

- **上游**：各角色團隊把產出寫進 Drive，本 skill 是它們的「共同閱覽窗口」。
- **下游**：讀到的客服報告 → `review-miner`；競品/情報報告 → `competitor-comparison`；供應鏈/成本訊號 → `financial-analyst`。
- **調度**：要把「讀到的多份報告」變成當日行動計畫，交 `hub-router` 統籌。

> 提醒：本 skill 只做「瀏覽 / 讀取 / 彙整」，不替報告內容背書、不產生新報告；任何無法從檔案內容直接讀到的推論一律標 `【需查證】`，並附可回溯的來源檔名與連結。
